# 프로젝트 규칙 (Project Rules)

이 파일은 프로젝트 진행 및 협업 시 준수해야 할 규칙을 정의합니다.

## 1. 언어 (Language)
*   **모든 대화 및 커뮤니케이션은 한국어(Korean)를 사용합니다.**
*   코드 내의 주석(Comment)은 한국어로 작성하는 것을 권장합니다.
*   문서(Documentation) 작성 시 한국어를 기본으로 하되, 필요한 경우 영어를 병기할 수 있습니다.

## 2. 코드 스타일 (Code Style)
*   기존 프로젝트의 코딩 컨벤션을 따릅니다.
*   변수명, 함수명 등은 영문으로 작성하되, 의미가 명확해야 합니다.

## 3. 문서화 (Documentation)
*   새로운 기능을 추가하거나 변경할 때는 반드시 관련 문서를 업데이트합니다.
*   복잡한 로직이나 아키텍처 변경 시에는 다이어그램 등을 활용하여 시각화합니다.
